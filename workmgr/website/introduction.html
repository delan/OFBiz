Introduction to The Open For Business Project

Project on Source Forge: The Open For Business Project (ofbiz)

Founders and Lead Architects: David E. Jones & Andy Zeneski

The Open For Business Project is an open source project that provides tools for businesses based on Sun's J2EE standard. The project is intended to be open and used by a large group of people. Because of that it is licensed using the MIT Open Source License.

The over-arching vision of the Open For Business Project is to embrace and enable The 5 E's: 

  1. Ease of Cost
  2. Ease of Installation
  3. Ease of Customization
  4. Ease of Integration
  5. Ease of Use.

The tools and components available can shorten J2EE business project cycles by months through a well designed architecture, pre-built components, and a dynamic entity model based data source abstraction layer. The goal of the Open For Business project is to provide useful, standards compliant tools and applications. 

These standards will include: XML; J2EE; RuleML; WfMC; BPML; ebXML; GAAP & FASB; OMG's Workflow, GL, Party, Product, etc; ArapXML; and other leading standards that relate to the various tools, components and applications of Open For Business.

To help us provide a strong data model foundation for the project we are using entity designs from various standards bodies, and the The Data Model Resource Book, Revised Edition, Volumes 1 & 2 written by Len Silverston and published by John Wiley & Sons, Copyright 2001. These books have been an excellent reference and useful design guide for handling business oriented data. Open For Business uses a variation on the models in the book, but many concepts from the book are still recognizable, enough so that it is very worth it to buy these books.

In addition to using widely accepted and detailed standards, Open For Business is also using design and architecture patterns that represent best practices which have been used and improved over the years. These patterns make the software more reliable and flexible. They also make it easier for developers to customize and enhance the software because developers familiar with the patterns and their common usage will easily recognize the organization of the software and be able to find and modify the desired components quickly.

The patterns used in the project now include:

  • General Patterns: Interface, Factory, Singleton, Double Checked Locking, Composite, Facade, Mediator
  • Presentation Tier: Intercepting Filter, Front Controller, View Helper, Dispatcher View, Composite View
  • Business Tier: Business Delegate, Value Object, Composite Entity, Value Object Assembler, Service Locator, Data Transfer HashMap Generic Attribute Access
  • Integration Tier: Data Access Object

Many other common patterns will be used throughout the project as they apply and as they can be implemented. For more information about these patterns visit the Sun J2EE Patterns page and TheServerSide.com patterns resources page. Sun has published a book based on the work on the above references Sun J2EE Patterns page called "Core J2EE Patterns", which is an excellent reference. Information about some of the more general patterns can be found in the classic Gang of Four "Design Patterns" book (Addison Wesley), or the "Patterns in Java" books (Vols 1 & 2) by Mark Grand (Wiley).
